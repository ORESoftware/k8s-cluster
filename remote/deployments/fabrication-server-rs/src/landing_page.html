DD Fabrication Server

A Rust planning and review service for 3D printing, CNC mills and routers, lathes, sheet cutters, EDM, postprocess cells, and hybrid split/combine manufacturing.

What It Takes In

Submit fabrication intents, native CAD, mesh/model/slicer sources, machine profiles, existing G-code, CAM intermediate files such as APT/CLDATA, controller programs, or text job sheets.

What It Builds

The server creates draft design packages, export bundles, process plans, machine schedules, generated instructions, machine-code reviews, release packages, and job artifacts.

How It Learns

Outcomes feed bounded MDP, POMDP, DES, reward, and neural-policy evidence using the local des_engine crate. Learned preferences stay advisory until release gates clear.

How It Stays Safe

Machine-ready release is blocked until validation, simulation or dry-run review, controller/postprocessor checks, setup evidence, quality evidence, and operator or automation signoff are attached.

How It Works

The service turns a submitted fabrication goal into evidence-backed choices: it imports or generates geometry, chooses candidate machines and materials, decomposes or combines parts when a single process is risky, generates or reviews instructions, simulates and validates release blockers, then records outcomes so later jobs can learn from the route.

1. Discover
Use /fabrication/capabilities, /fabrication/intake/catalog, /fabrication/templates/catalog, schema, and examples.
2. Intake
Provide design inputs, instructions, machines, materials, and review evidence.
3. Plan
Generate route, split/combine, setup, toolpath, schedule, and release-package drafts.
4. Validate
Analyze imported or generated instructions for machine-failure, human-intervention, and release blockers.
5. Learn
Post outcomes and worker result reviews so future jobs can prefer safer strategies.
6. Inspect
Fetch retained jobs, artifacts, and release bundles before downstream release review.

Start Here

Most integrations follow the same evidence path: discover what the service supports, submit source geometry or instructions, ask for a draft plan or review, attach validation results, then feed the outcome back into learning.

  • 1. Discover
    Capabilities, machines, materials, and request templates.
  • 2. Import Or Generate
    Design import, design generation, instruction languages, and machine-code generation.
  • 3. Validate And Improve
    Validation, failure boundaries, remediation, and instruction improvement.
  • 4. Split, Combine, Or Release

Design And Toolchain Intake

Native and cloud CAD intake includes PTC Creo / Pro/ENGINEER, SOLIDWORKS, Autodesk Fusion, Siemens NX, CATIA, Onshape, FreeCAD, OpenSCAD, Blender, and ZBrush, plus neutral and print handoffs such as STEP, Parasolid, STL, 3MF, OBJ, AMF, and slicer projects from PrusaSlicer, OrcaSlicer, Cura, and Bambu Studio. Ambiguous extensions such as .prt and .asm require source-system or translator evidence before downstream release.

Release Gates

Generated designs, toolpaths, slicer plans, G-code, controller programs, and job-sheet interpretations stay advisory until the release packet proves each critical gate.

  • Source provenance
    CAD, mesh, CAM, controller, macro, and text instruction origin is identified.
  • Machine envelope
    Axes, fixtures, work offsets, tools, support media, and controller modes are verified.
  • Process readiness
    Thermal, spindle, feed, coolant, dust, gas, abrasive, resin, powder, or filament state is ready.
  • Simulation evidence
    Dry-run, collision, reach, support, quality, and postprocess reviews clear blockers.
  • Human or automation handoff
    Required operator interventions, split/combine joins, and restart steps are signed off.
  • Learning disposition
    MDP/POMDP/DES/neural recommendations remain advisory until retained outcomes support promotion.

Priority Dispositions

Worker result routes publish priorityDispositions so review workers can see which evidence lanes are blocked, need review, are closed, or are ready for learning. Blocked and pending-blocker-resolution dispositions keep machineReady false until the matching release gate has retained evidence.
